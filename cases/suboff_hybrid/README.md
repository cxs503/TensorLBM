# SUBOFF Hybrid DG-LBM Case

This case runs a DARPA SUBOFF-inspired submarine hull inside a rectangular
channel using the **DG-LBM hybrid solver**:

* **Near-wall DG zone** – cells within `dg_band` lattice units of the hull
  surface use the first-order Chapman–Enskog DG collision operator, which
  reconstructs the strain-rate tensor from explicit velocity gradients.
* **LBM exterior** – the remaining fluid cells use the standard D3Q19 BGK
  collision.

The solver reuses the same coupling infrastructure as the merged sphere
DG-LBM case (PR #132).

---

## Quick-start

> All commands should be run from the **repository root**.

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Smoke run (coarse grid, ~seconds on CPU)

```bash
PYTHONPATH=src python cases/suboff_hybrid/run_suboff_hybrid.py \
    --nx 80 --ny 40 --nz 40 \
    --hull-length 48 --re 200 \
    --n-steps 20 --output-interval 10 \
    --run-name smoke --overwrite
```

### 3. Production run (CPU, ~minutes)

```bash
PYTHONPATH=src python cases/suboff_hybrid/run_suboff_hybrid.py \
    --nx 200 --ny 80 --nz 80 \
    --hull-length 120 --re 200 \
    --n-steps 500 --output-interval 100 \
    --run-name suboff_re200 --overwrite
```

### 4. Post-process

```bash
PYTHONPATH=src python cases/suboff_hybrid/postprocess_suboff_hybrid.py \
    --run-dir outputs/dg_lbm_suboff/suboff_re200
```

| Parameter | CLI flag | Default | Description |
|---|---|---|---|
| Grid (x) | `--nx` | 200 | Axial (streamwise) cells |
| Grid (y) | `--ny` | 80 | Transverse cells |

## Architecture notes

### Geometry

The SUBOFF bare-hull mask is generated by `build_suboff_mask` from
`src/tensorlbm/suboff_cad.py`.  The near-wall DG zone is built by
`build_dg_hull_band_mask` (in `src/tensorlbm/dg_lbm.py`), which dilates
the solid mask using a 3-D max-pool operation with radius `ceil(dg_band)`.

### Coupling

The DG↔LBM coupling is handled transparently by `collide_dg_lbm` — the
same function used for the sphere case — via a Boolean `dg_mask` tensor:

```
f_out = collide_dg_lbm(f, tau=tau, dg_mask=dg_mask)
```

No interface flux exchange or ghost-cell communication is required; the
mask-gated collision naturally blends the two operators at every step.

### Stability guidelines

* Keep `tau > 0.5` (i.e. `nu = u_in * hull_length / re > 0`).
* Recommended `u_in ≤ 0.08` to limit compressibility errors.
* Increase `dg_band` (e.g. to 6–8 lu) for finer near-wall resolution at
  higher Re.

---

## Known limitations

* The DG operator is first-order (linear Chapman–Enskog); higher-order
  polynomial reconstruction is reserved for future work (`dg_order=1` only).
* Drag/force integrals are not yet computed inline; use the existing
  `compute_obstacle_forces_3d` utility from `suboff_resistance.py` for
  post-hoc force extraction.
* Appendages (sail, fins) supported at geometry level; additional resolution
  may be needed around fin tips at high Re.
